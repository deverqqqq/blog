必要

  1. 熱情

    從零開始學習，一定會遇到很多的挫折，沒有熱情的話是絕對撐不下去的，但只要感興趣，一點一點慢慢做，一定可以隨著自己的進步得到更多成就感。

  2. Git

    就算沒有從事程式設計，我覺得學習 Git 的知識來管理 markdown 文件的內容，是再好不過的選擇。因為整個這個系列是採用 Github 作為放置檔案的站點，完全不會 Git 的話基本上就不能考慮這個做法了。(PS: 很多主題也是在 github 提供問題回饋跟更新的發布 )

    Git 是一套分散式的版本控管系統，入門容易精通難，但是他的設計理念，是非常聰明、先進，而且實用的。

有的話更好，不會也沒關係

  1. Html + CSS

    網頁呈現的基本標籤，如果要動手修改主題的話，至少要知道基本標籤的含意與用法，不然會無從下手，但如果只是要套用現有的完整主題，其實倒也不是那麼必要。

  2. Python + Javascript

    因為本站的圖片另外放在另外一個站點，加上有作一些訪問的處理，需要修改官方提供的 python 跟 javascript 內容，來滿足自己的需求。( 其實如果全部放在一起倒也不用那麼麻煩，架在 firebase 上應該也不會太困難，只是考慮到其他因素，最後我還是決定要拆兩個站點，之後會在後面的文章分緣由。)
